关于Unix(Linux)系统下的/usr文件夹

2016-05-17 • 分类: Linux • 标签: linux

Unix(Linux)下的/usr文件夹，我一直以为是”user”的缩写，而且周围的很多人也跟我是同样的观点，也都读作”user”。

后来想完善下自己的linux知识，看《鸟哥的linux私房菜》的时候，书上说，/usr文件夹其实是”Unix System Resources”的缩写，作用类似于Window下的”Program Files”文件夹。一想也是，用户的文件不是都放在/home文件夹下面的吗？于是恍然大悟，一面大呼”得之矣，得之矣”，一面悔恨自己学知识太不求甚解了，作为技术人员，此乃大忌。

岂料后来剧情又发生翻转，又看到的很多资料上，包括类似wiki的权威资料上面，都多少把/usr看做是用户文件夹，比如：

/usr usually contains by far the largest share of data on a system. Hence, this is one of the most important directories in the system as it contains all the user binaries, their documentation, libraries, header files, etc…. X and its supporting libraries can be found here.

再细究起来，才发现，这原来是Unix系统的历史问题造成的。

原来最初Unix的存储盘容量很小，随着Unix系统目录变得越来越大，一个存储盘已经不够用，于是添加了第二块存储盘，专门用来存放用户的数据，挂载点即为/usr，而且两块存储盘上的目录结构是一模一样的。再后来，又添加了第三块存储盘，规定第三块存储盘的挂载点为/home，存放用户的数据，第二块存储盘上存放用户的程序。

这种存储结构后来一直延续了下来，只是/usr文件夹不再只是存储用户的程序，而是也包括系统开发商开发的程序，用户的程序存放在/usr/local目录下。

所以说，开始的两种说法都只说对了一部分，都不全面。

其实，因为历史原因导致的概念不清不止这一个，还有比如回车和换行的区别。具体的说明，可以参看阮一峰的博客，本文也参考了阮老师的博文。
